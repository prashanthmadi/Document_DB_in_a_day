# Module 2: NoSQL Core Concepts

**Duration:** 3-4.5 hours | **Levels:** L100 → L200 → L300

This module covers everything you need to know about NoSQL fundamentals, schema design, and data modeling for DocumentDB.

---

## 📚 Module Structure

### [L100: NoSQL Fundamentals](L100_nosql_fundamentals.md) (60-90 minutes)
**Level:** Beginner | **Hands-on:** ✅

Learn the basics of NoSQL and DocumentDB querying:
- Why NoSQL? Understanding document databases
- Basic querying with `find()`
- Query operators and filters
- Aggregation pipelines
- Hands-on exercises

**Prerequisites:** Module 1 completed

---

### [L200: Schema Design Patterns](L200_schema_design_patterns.md) (60-90 minutes)
**Level:** Intermediate | **Hands-on:** ✅

Master common design patterns and anti-patterns:
- Embed vs Reference decisions
- 5 Key Patterns: Computed, Subset, Extended Reference, Bucket, Outlier
- 5 Anti-Patterns to Avoid
- Blog platform design lab
- Real-world pattern application

**Prerequisites:** L100 completed

---

### [L300: Data Relationships](L300_data_relationships.md) (45-60 minutes)
**Level:** Advanced | **Hands-on:** ✅

Model complex relationships in NoSQL:
- One-to-one relationships
- One-to-many relationships
- Many-to-many relationships
- Join collections
- Real-world scenarios (e-commerce, social media)

**Prerequisites:** L100 & L200 completed
